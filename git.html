Git Cheatsheet

Basics

git init
- initialize git repository 

git add <FILE>
- add file to staging area - to be tracked by git. 

git commit -m "commit message"
- commit changes, must provide commit message (or default git editor will be opened) 

git commit -am "commit message"
- add and commit with one command

Branching

git branch
- show actual branch name

git branch -M main
- rename actual branch for arg, here: main

git checkout -b feature
- create new branch called feature and checkout to it

Typical flow

  1. We do some work on feature branch ...
  2. We now want to merge them, add the changes, to the master (or recently more popular main)
  3. git checkout master
    - we switch to master branch
  4. git merge feature
    - we merge from branch provided as argument, here feature
  • Sometimes there can be a conflict, so then we have to resolve it manually. VS Code can show us current and incoming changes in a appealing way.
